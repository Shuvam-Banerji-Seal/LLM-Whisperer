KEY FINDINGS AND INSIGHTS
================================================================================

ALGORITHMIC SELECTION:
  • HNSW optimal for: <100M vectors, latency-critical (<5ms)
  • IVF optimal for: >100M vectors, throughput-critical (>10K QPS)
  • GPU acceleration (CAGRA): Best for >50M vectors on GPU
  • NSG: Emerging alternative, less widely deployed

QUANTIZATION IMPACT:
  • Scalar (4-bit): 8x compression, 2x speedup, <1% recall loss ★ RECOMMENDED
  • Binary (1-bit): 32x compression, 40x speedup, 5-10% loss (recoverable)
  • Standard deployment: Always include quantization at scale

PERFORMANCE TRADEOFFS:
  
  Metric           HNSW              IVF-Flat         IVF-PQ
  ────────────────────────────────────────────────────────────
  Latency (1 query) 5-10ms          10-50ms          5-20ms
  Throughput      1K-10K QPS        50K QPS          100K-500K QPS
  Memory (100M)   30 GB             50 GB            10 GB
  Index Build     1-2 µs/vector     0.1 µs/vector    0.5 µs/vector
  Approximate Cost $5-10K            $2-3K            $1-2K
  
  ★ Best single-query: HNSW
  ★ Best batch throughput: IVF-PQ
  ★ Best cost/performance: IVF-PQ with quantization

PRODUCTION INSIGHTS:
  1. Quantization is production standard (not optional)
  2. Sharding required for >100M vectors
  3. GPU acceleration ROI: ~50M vectors
  4. Batch processing dominates at scale (10K+QPS)
  5. Reranking recovers accuracy effectively

BILLION-SCALE ECONOMICS:
  • HNSW approach: $15-20K infrastructure, too much memory
  • IVF-PQ approach: $3-5K infrastructure, 4-5x cost reduction
  • With GPU: Add $10-30K but achieve 2-4x throughput
